## The pitch

Claude Code is the terminal-native harness Anthropic ships to drive its models. This review is about
that software, not the models underneath it (those earn their own verdict) and not the skills,
agents, and workflows you bolt on top (each its own thing). What you are actually buying is the
harness: the agent loop, the memory, the configuration surface, and the extension points. Judge it
on how well it lets a careful human aim a capable model, because that is the job it has.

## The memory is the part to steal

The standout is the memory. Claude Code keeps its project memory in plain files, and that simplicity
is the whole win, because plain files go into git. Point `autoMemoryDirectory` at a committed
`.claude/memory` folder and keep the machine-specific `settings.local.json` out of version control.
Every developer who clones the repo then inherits the same hard-won lessons instead of relearning
them one machine at a time. I wrote up the exact setup
[separately](https://medium.com/@ajmeese7/version-controlling-your-claude-memory-for-projects-e91c8c47cb06).
It sits alongside `CLAUDE.md` and your existing docs rather than replacing them. The division of
labor is right: the docs explain the system, the memory remembers what went wrong last time.

One caveat is worth planning for. The memory and config are Claude-shaped, so a repo that other
agents also touch should hand them the same context, for example by symlinking an
[`AGENTS.md`](https://agents.md) so a Codex or Copilot session is not flying blind. Lock your
accumulated intelligence to one vendor and you have only traded one dependency for another.

## A platform, not a CLI

Underneath the prompt, Claude Code is extensible in a way the stock experience does not advertise.
Skills, hooks, MCP servers, and subagents turn it from a chat in a terminal into something you shape
around your own workflow. The individual pieces you build on it are their own reviews; what matters
here is that the platform is open enough to build them at all. I avoid the stock setup when I can,
but running it bare is exactly what surfaced the gains worth chasing: the token waste and the
missing guardrails that a few hooks and a tight `CLAUDE.md` clean up. The floor is fine. The ceiling
is the reason to stay.

## Where it bites

It does not run itself, and anyone selling you a hands-off loop is selling you a maintenance bill.
Stop steering and stop tending the memory and config, and it drifts. It ignores instructions, takes
the lazy path, and produces output that looks productive while quietly eroding the maintainability
and performance of whatever you are building. The fix is not a better prompt, it is a human who
stays in the loop and a plan written before the agent starts. That plan discipline is not specific
to Claude Code, but the failure mode it prevents shows up here as fast as anywhere: undefined scope,
confident output, long-term mess. None of this knocks it far, because the competition has the same
problem and less to work with. It only means the leverage is real when you supply the judgment.

## Against the field

I have driven the alternatives. [Cursor](https://cursor.com) chains you to the IDE, and I would
rather steer an agent from anywhere than sit at an editor. [Codex](https://github.com/openai/codex)
I still use, though through my own
[Hermes](https://ajmeese7.medium.com/how-i-set-up-multi-user-ai-agents-with-hermes-and-you-can-too-648b52fe20e9)
agent over Telegram rather than the command line, since steering it from a chat thread beats sitting
at a prompt. For serious work, Claude Code is what I reach for, from fine-tuning a resume against a
job posting out of a knowledge base to building enterprise-scale platforms that are not public yet.
It blows the field away today, and with Anthropic's resources behind it that lead is not likely to
shrink.

## What it costs

It costs real money. The $200/month plan is what carries serious parallel work; the $100 tier is
fine for light use, but the limits start getting in the way the moment you run anything complex in
parallel. It is a steep number to pay, and the quality gap over everything else still makes it the
easy call.
